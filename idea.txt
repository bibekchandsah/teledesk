        User B (receiver) sees a "User A has left the call" message in the call status area, and the call ends for both users.

    If user B (receiver) closes the call window:
        User A sees a "User B has left the call" message in the call status area, and the call ends for both users.
        User B's client sends a "call ended" signal to the backend before closing, so the backend can notify User A immediately.
    In both cases, if the client fails to send the signal (e.g. crash or network loss), the backend detects the disconnection via WebSocket and notifies the other user accordingly.
