FIXED: 2FA Verification Modal Showing on Page Reload

Issue: After successfully logging in with 2FA, the verification modal appears again on every page reload.

Root Cause:
- The 2FA check was running on every auth state change (including page reloads)
- useRef doesn't persist across page reloads (it's in-memory only)
- Firebase auth persists across page reloads, triggering onAuthChange again
- Each page reload would check if 2FA is enabled and show the modal again

Solution Applied:
- Use `sessionStorage` instead of `useRef` to persist 2FA verification status
- sessionStorage persists across page reloads but clears when browser tab/window closes
- Check sessionStorage before showing 2FA modal
- Clear sessionStorage on logout

Implementation:
1. Created helper functions:
   - `is2FAVerifiedInSession(uid)` - Check if UID is verified in sessionStorage
   - `mark2FAVerified(uid)` - Store verified UID in sessionStorage
   - `clear2FAVerification()` - Remove verification from sessionStorage

2. Check `is2FAVerifiedInSession(fbUser.uid)` before showing 2FA modal
3. Call `mark2FAVerified(pending2FAUser.uid)` on successful verification
4. Call `clear2FAVerification()` on logout and auth state null

Files Modified:
- desktop-client/src/context/AuthContext.tsx

Result:
- 2FA verification is now only required once per browser session
- Page reloads no longer trigger 2FA modal
- User stays logged in until they explicitly log out or close the browser tab
- Logging out or closing tab clears the session and requires 2FA verification again on next login
- sessionStorage is more secure than localStorage as it clears when tab closes
