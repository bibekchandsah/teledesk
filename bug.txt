FIXED: Chat Theme Blur and Opacity Effects + Instant Peer Sync

Issue 1: Blur and opacity effects were not working properly in the chat window.

Root Cause:
- The backdrop-filter blur effect requires proper DOM layering
- The overlay was positioned as a sibling to messages, not as a parent wrapper
- This prevented the backdrop-filter from applying correctly to the background

Solution Applied:
1. Restructured the messages container DOM hierarchy:
   - Background image on outer scroll container
   - Wrapper div with relative positioning
   - Backdrop overlay with absolute positioning (opacity + blur)
   - Messages content with relative positioning and z-index

2. Updated ChatThemeModal preview to match the same structure

3. Added conditional blur application (only when blur > 0)

Issue 2: Opposite user unable to see theme changes instantly when "Show to Others" is enabled.

Root Causes:
1. Backend was emitting to `peerUid` instead of `user:${peerUid}` room
2. Frontend was using `useChatStore.getState()` instead of the hook, preventing re-renders
3. Missing `setUserProfile` in useEffect dependencies

Solution Applied:
1. Fixed backend socket emissions to use proper room names:
   - Changed `_io.to(uid)` to `_io.to(\`user:${uid}\`)`
   - Changed `_io.to(peerUid)` to `_io.to(\`user:${peerUid}\`)`

2. Fixed frontend socket listener:
   - Added `setUserProfile` to useChatStore destructuring
   - Removed `useChatStore.getState()` calls in handlers
   - Added `setUserProfile` to useEffect dependencies
